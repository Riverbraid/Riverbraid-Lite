# Contributing to Riverbraid

Thank you for your interest in the Riverbraid architecture. Because this cluster is built on the principles of **Stationary Intelligence**, we do not follow a "move fast and break things" model. 

### 🛡️ The Governance Lock
Every repository in this organization is governed by the Absolute V2 standard. Any proposed changes must:
1. **Maintain Determinism:** We reject all "black box" or non-deterministic logic.
2. **Respect the Braid:** Changes to one petal must be checked against the collective Merkle Root.
3. **Primary Meaning:** We prioritize rule-and-anchor logic over raw data volume.

### 🛠️ Submission Process
1. **RFC First:** Open an issue to discuss the intent of the change.
2. **Audit Check:** All Pull Requests must pass the `CI` verification suite.
3. **Stationary Sealing:** Approved changes will result in a new cluster-wide version tag (v1.x.x).
